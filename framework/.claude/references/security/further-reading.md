# Security — Further Reading

Use these as starting points for deeper research. Use WebSearch for further research as required.

## OWASP

- [OWASP Cheat Sheet Series](https://github.com/OWASP/CheatSheetSeries/tree/master/cheatsheets) — practical, topic-by-topic implementation guides (auth, input validation, XSS, SQL injection, secrets, cryptography, and 80+ more)
- [OWASP Web Top 10 (2021)](https://owasp.org/www-project-top-ten/) — canonical web application risk list
- [OWASP API Security Top 10 (2023)](https://owasp.org/API-Security/) — API-specific risks
- [OWASP LLM Top 10 (2025)](https://genai.owasp.org/resource/owasp-top-10-for-llm-applications-2025/) — LLM application risks
- [OWASP Agentic AI Top 10 (2026)](https://genai.owasp.org/resource/owasp-top-10-for-agentic-applications-for-2026/) — autonomous agent risks
- [OWASP MCP Top 10 (2025)](https://owasp.org/www-project-mcp-top-10/) — Model Context Protocol risks

## Standards & Frameworks

- [CWE Top 25](https://cwe.mitre.org/top25/) — most dangerous software weaknesses at code level
- [STRIDE Threat Modelling](https://learn.microsoft.com/en-us/azure/security/develop/threat-modeling-tool-threats) — Microsoft's threat modelling primer
- [AWS Well-Architected Framework - Security Pillar](https://github.com/aws-samples/sample-well-architected-skills-and-steering/blob/main/skills/security-assessment/references/security-questions.md)
- [Azure Well-Architected Framework - Security Pillar](https://learn.microsoft.com/en-us/azure/well-architected/security/checklist)
- [GCP Well-Architected Framework - Security Pillar](https://docs.cloud.google.com/architecture/framework/security)
